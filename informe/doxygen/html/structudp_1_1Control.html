Detailed Description

This structures handles the control comunication between the application layer and the UDP Protocol.

Author
Laouen Louan Mayal Belloli
Date
14 May 2017

Every command as connect, bind, read, etc are requested from the application layer using an udp::Control struct, specifying the socket and application ID. An udp::Control has the next fields:

  1. local_port: The local unsigned short port of the socket that the control message refers to.
  2. remote_port: The remote unsigned short port of the socket that the control message refers to (only for a connected socket).
  3. local_ip: The local IPv4 ip of the socket that the control message refers to.
  4. remote_ip: The remote IPv4 ip of the socket that the control message refers to (only for a connected socket).
  5. app_id: The application ID bound to the refered socket.
  6. request: The udp::Ctrl with the request command.
  7. packet: If the request command is send/write/send_to/write_to, this fields must have the dns::Packet to send through the network.

bool udp::Control::isAppRequest ( ) const
inline

Checks whether the udp::Control is a request from the appplication layer to the UDP protocol or not.

A request can be something else than a command from the application layer to be excecuted by the UDP protocol, as for example the udp::Ctrl::INVALID_SOCKET request that is a control messages to tell the application layer the specified socket is invalid for the send command.

Returns
True if it is an application request, False otherwise.
