\subsection{Detailed Description}
This structures handles the control comunication between the application layer and the U\+DP \hyperlink{classProtocol}{Protocol}. 

\begin{DoxyAuthor}{Author}
Laouen Louan Mayal Belloli 
\end{DoxyAuthor}
\begin{DoxyDate}{Date}
14 May 2017
\end{DoxyDate}
Every command as connect, bind, read, etc are requested from the application layer using an \hyperlink{structudp_1_1Control}{udp\+::\+Control} struct, specifying the socket and application ID. An \hyperlink{structudp_1_1Control}{udp\+::\+Control} has the next fields\+:
\begin{DoxyEnumerate}
\item local\+\_\+port\+: The local unsigned short port of the socket that the control message refers to.
\item remote\+\_\+port\+: The remote unsigned short port of the socket that the control message refers to (only for a connected socket).
\item local\+\_\+ip\+: The local \hyperlink{structIPv4}{I\+Pv4} ip of the socket that the control message refers to.
\item remote\+\_\+ip\+: The remote \hyperlink{structIPv4}{I\+Pv4} ip of the socket that the control message refers to (only for a connected socket).
\item app\+\_\+id\+: The application ID bound to the refered socket.
\item request\+: The udp\+::\+Ctrl with the request command.
\item packet\+: If the request command is send/write/send\+\_\+to/write\+\_\+to, this fields must have the \hyperlink{structdns_1_1Packet}{dns\+::\+Packet} to send through the network. 
\end{DoxyEnumerate}
